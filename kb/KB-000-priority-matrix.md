# KB-000: Ticket Priority Matrix — Summit Managed IT

Priority is judged from **impact x urgency**, independent of category.

| Priority | Definition | Examples |
|---|---|---|
| critical | Site-wide outage; active security incident; executive blocked on same-day revenue/legal deadline; involuntary termination access revocation | Server down for office; ransomware; CEO blocked during a closing |
| high | Multiple users blocked; single user blocked with hard same-day deadline; recurring data-loss risk | Team loses drive access; daily BSODs |
| medium | Single user degraded or blocked, no hard deadline | Lockout, VPN failure, printer offline |
| low | Inconvenience, request, or question; workaround exists | Software request, storage warning, slow wifi |

Modifiers: executive sender +1 level when business impact stated; explicit same-day deadline +1 level; multiple users affected +1 level. Security category is always high or critical.
